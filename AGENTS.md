1) Команды сборки, линтинга и тестирования
- Локальный запуск приложения (без Docker):
  - Установить зависимости: python -m venv venv && source venv/bin/activate && pip install -r requirements.txt
  - Запуск сервиса разработки: uvicorn backend.app.main:app --reload --port 8000
  - Тесты: pytest -q
- Запуск тестов по одному тесту:
  - pytest tests/test_module.py::TestClass::test_method -q
  - или: pytest -k "test_method" tests/test_module.py -q
- Линтинг и форматирование:
  - Ruff для статического анализа: ruff check .
  - Ruff автоформатирование: ruff format .
  - isort для импорта: isort .
  - Black для форматирования: black .
- Контроль типов (опционально): mypy . --ignore-mithril-errors (при наличии конфигурации)
- Сборка и запуск через Docker Compose:
  - docker compose up -d --build
  - docker compose ps
  - docker compose logs -f
- Вариант Blues/CI (при наличии):
  -CI: запускается через ваш CI-сервер, например GitHub Actions; убедитесь, что шаги: install, lint, test, typecheck, build.

2) Руководство по стилю кода
- Общие принципы:
  - Язык: Python 3.11+; совместимость с текущим окружением.
  - Форматирование: Black (строки 88 символов; пробелы 4; табы запрещены).
  - Импорт-порядок: стандартная библиотека; сторонние пакеты; локальные модули. Каждая группа разделена пустой строкой; используйте isort для авто‑упорядочивания.
  - Типизация: аннотации типов повсеместно; Optional, List, Dict, Tuple, Union; from __future__ import annotations.
  - Докстринги: используйте Google‑style или NumPy‑style, но оставайтесь единообразными в проекте.
  - Названия: функции/переменные — snake_case; классы — CamelCase; константы — ALL_CAPS.
  - Ошибки: избегайте bare except; ловите конкретные исключения; добавляйте контекст в исключения.
  - Логирование: вместо print используйте logging; не выводите чувствительные данные в логи.
  - Документация: модули и функции должны иметь краткие docstrings.
  - Асинхронность: при использовании asyncio — валидируйте корректное использование await/async; избегайте блокирующих операций без нужна.
  - Безопасность: не зашивайте секреты в код; используйте переменные окружения и секрет‑хранилища.

- Правила именования and структура импорта:
  - Импорты внутри файла: сначала стандартная библиотека, затем сторонние, затем локальные.
  - Не используйте wildcard импорты (from module import *).
  - Импорты группируются по логике модуля и упорядочиваются средствами isort.

- Обработка ошибок и границы:
  - Всегда возвращайте понятные коды ошибок или исключения; не прячьте исключения за общий Exception.
  - На границе API возвращайте структурированные ошибки: код, сообщение, контекст, возможные remediation.
- Ответственности и тесты:
  - Каждый метод/функция должен иметь unit-тесты; тесты должны покрывать обычные и крайние случаи.
  - Тесты должны быть независимы и повторяемы.

- Стандартные сигнатуры и соглашения:
  - API‑модели: Pydantic (если используется) или dataclasses; валидируйте входы и выходы.
  - Внутренние сервисы должны возвращать валидные типы и исключения, которые ловит верхний уровень.

22) Windows/Docker Desktop (WSL2) deployment notes
- Подход: развёртывать локально на Windows через Docker Desktop с WSL2 интеграцией.
- Требования: Docker Desktop для Windows, включён WSL2 (Linux дистрибутив в WSL), достаточно оперативной памяти (8–16 ГБ).
- Старт стека:
  - В командной строке PowerShell запустите: docker compose up -d --build
- Сетевые настройки:
  - Убедитесь, что порт 8000 доступен из локальной сети.
- Доступ к данным:
  - Для монтирования директорий под индексацию используйте bind‑mountы на Windows, например:
    - в вашем docker-compose.yml добавьте volumes: - "C:/data/docs:/data:ro"
- Тестирование:
  - После запуска: "curl http://localhost:8000/health" в PowerShell.
- Примечание: Linux‑контейнеры работают под управлением WSL2; используйте Windows‑посредник только для UI/CLI.
- Пример использования Windows override:
- docker compose -f docker-compose.yml -f docker-compose.windows.yml up -d --build
- Убедиться, что данные в C:/data/docs и C:/data/db доступны и примонтированы как read-only (ro).
